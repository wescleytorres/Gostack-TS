=> Tabela de relacionamento BD

-> Um para Um (OneToOne)
  Um usuario tem um agendamento.

-> Um para Muitos (OneToMany)
  Um usuario tem muitos agendamentos.

-> Muitos para Muitos (ManyToMany)
  Muitos usuarios participam do mesmo agendamento.
